# 公告证据合同与因果边界

## 证据层级

- L0 元数据：代码、公告 ID、披露时间、来源。
- L1 标题：只能支持主题分类，例如“疑似回购公告”。
- L2 正文/PDF：支持金额、比例、期限、条件、交易对手、完成状态和风险条款。

涉及数值或影响结论时至少需要 L2。只有 L1 时输出 `needs_full_text=true`，不要补写正文事实。

## 确定性分类

允许的主题标签包括：

- `earnings`：业绩预告、快报、定期报告；
- `dividend`：分红、派息、送转；
- `repurchase`：回购；
- `holding_change`：增持、减持、解禁；
- `restructuring`：并购、重组、资产交易；
- `litigation_regulatory`：诉讼、处罚、立案、问询；
- `suspension`：停牌、复牌；
- `financing`：定增、配股、可转债、担保、质押；
- `governance`：高管变动、股东大会、章程；
- `other`：标题不足或未命中。

关键词命中只产生 taxonomy，不产生 `positive`/`negative` 情绪标签。

## 时间与去重

- 优先使用公告 ID 或规范化 URL 去重；缺失时使用 `notice_date + 规范化标题`。
- `notice_date` 只有日期而无时分秒时，不能断言是盘前还是盘后。
- 价格归因需把公告可获知时间映射到下一可交易时段，并同时检查指数、行业和其他事件。
- 同一事项的提示性公告、进展公告和完成公告是不同阶段，不应仅因标题相似全部删除。

## 风险措辞

使用“可能相关”“时间邻近”“需正文确认”“未证明因果”。不要使用“必然利好”“导致上涨”等确定性措辞，除非有独立、充分证据。

## 失败模式

| 情况 | 处理 |
| --- | --- |
| 无 title | error/排除并 warning |
| 无日期 | 保留但放到未定时间组 |
| 无 URL/PDF | 标记证据不可下钻 |
| 只有标题却提取金额 | 拒绝 |
| 同标题不同日期/阶段 | 不自动合并 |
| 把关键词分类当影响方向 | 拒绝 |
