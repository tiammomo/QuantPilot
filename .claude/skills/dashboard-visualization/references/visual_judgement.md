# QuantPilot 可视化判读清单

本文件用于 `dashboard-visualization` 在动手写页面前做数据和图表判读。它不是通用数据可视化 pipeline，也不要求额外生成 judgement 文件；目标是在 QuantPilot 的 `run_plan`、final 数据和 evidence 已存在的前提下，先确认“该画什么、哪些指标能怎么算、哪些图表不能乱用”。

## 判读顺序

1. 先读 `.quantpilot/run_plan.json`、`data_file/final/dashboard-data.json`、`evidence/sources.json` 和 `evidence/data_quality.json`。
2. 列出本次页面的：
   - 场景模板族和变体：来自 `visualization.templateId/template_id`、`variantId/variant_id`。
   - 时间字段：如 `date`、`trade_date`、`report_date`、`quote_time`、`as_of`。
   - 维度字段：如 `symbol`、`name`、`asset_type`、`capability`、`period`、`source`。
   - 指标字段：如 `price`、`change_percent`、`close`、`amount`、`turnover`、`period_return`、`max_drawdown`、`roe`。
   - 可视化主语：单标的、候选池、多标的对比、组合持仓、回测曲线或财务报告期。
3. 判断每个核心指标的口径：它是快照值、可累加值、比率、收益率、风险指标、排名分数还是文本/状态。
4. 先确定首屏必备组件，再写页面。首屏组件应来自场景痛点，不从“好看的通用卡片”开始。

## 金融指标口径

常见字段的处理原则：

- 价格、昨收、开高低收：快照或时序点位，不能跨天求和。
- 成交量、成交额：可在同一窗口内求和或均值，但必须说明窗口和频率。
- 涨跌幅、收益率、换手率、毛利率、净利率、ROE：比率类指标，不能把每日/报告期比率直接相加。
- 区间收益：应由起止价格或标准字段计算，不能把日涨跌幅简单相加冒充收益。
- 最大回撤：从净值或收盘价序列计算，越小通常越好；展示时要标明为回撤而非普通收益。
- 波动率：必须说明窗口、日线/月线和是否年化。
- MA、量能比、趋势模板：必须来自同一复权口径和同一 K 线窗口。
- 财务报告期指标：必须保留报告期，不能混合 TTM、年度和季度口径后直接排序。
- 组合权重、仓位、集中度：分母必须是组合总市值或总资产，不能用样本条数替代。

卡片或摘要里涉及比率、留存、收益、回撤、胜率、命中率、仓位等口径敏感指标时，说明要回答四件事：时间窗口、覆盖对象、计算方法、样本数量或对比对象。不要裸写内部字段名或公式。

## 图表选择

按数据形态选图：

- 单标的 K 线：OHLC/K 线主图 + 成交量副图 + MA 图例 + 最近数据表。
- 多标的对比：统一尺度的收益/回撤/波动矩阵或柱状对比，不能只放多张小 sparkline。
- 组合持仓：持仓矩阵、权重/集中度、盈亏偏离和风险暴露，比普通股票列表优先级更高。
- 财务报告期：营收/利润趋势、ROE/毛利率趋势和报告期表格。
- 回测：净值曲线、回撤曲线、交易明细和参数卡。
- 事件/公告：时间线或表格，必须能回到公告时间和来源。

禁用或慎用：

- 时间序列不要切成饼图。
- 排名图不要只画相对宽度而没有数值标签和统一基线。
- K 线缩略图不能替代主图。
- 缺数据时不要删除图表区域；保留专业空态和下一步取数线索。

## 组件文案

用户可见文案要翻译金融口径：

- 好：`近 120 个交易日按前复权收盘价计算，区间收益为期末收盘价相对期初收盘价的变化。`
- 好：`最大回撤基于净值曲线的历史高点回落幅度，数值越大代表期间回撤越深。`
- 差：`period_return = close[-1] / close[0] - 1`
- 差：`retained_cnt / base_cnt`
- 差：`样本口径：field_analysis.metric_policies`

技术字段可以在 evidence 或数据质量区域保留，但不要作为指标卡主文案。

## 自检

提交前至少确认：

- `visualization.required_components` 中的核心组件已渲染或在缺口区说明。
- 图表和表格使用同一时间窗口、同一复权口径和同一标的集合。
- 卡片的比率、收益、回撤、胜率等描述没有把不可累加指标当成可累加值。
- 页面不依赖外部 CDN、远程 API、mock/static 数据或浏览器跨域直连。
- 主图、矩阵或表格有坐标/标签/tooltip/数值 fallback 中至少一种可读机制。
- 移动端没有页面级横向溢出，长表格只在内部滚动。
