# 财务报告口径合同

## 必需字段

每个报告至少保留：

- `report_date`：会计期间结束日，推荐 `YYYY-MM-DD`。
- `notice_date`：披露日；做 point-in-time 分析时必需。
- `data_type`：年度、半年度、季度累计或单季等口径。
- 原始来源及金额单位。

常用指标包括 `revenue`、`parent_net_profit`、`basic_eps`、`weighted_roe`、`gross_margin`、`revenue_yoy`、`net_profit_yoy`。字段缺失用 `null`，不要用零占位。

## 期间语义

- A 股一季报、半年报、三季报中的利润表指标常为年初至报告期末累计值；没有明确单季字段时，不称为单季收入/利润。
- 从累计值推单季值必须具备同年度、相同会计口径的相邻累计报告，并明确公式与重述风险。
- 年报与同年度三季报的差值可作为四季度近似值，但仍需标记 `derived`，不能冒充原始披露。
- 同比增长应优先使用上游已披露字段；自行计算时要求同期间、同币种、同会计口径，且基期不为零。

## 披露与重述

- 按 `report_date` 分组；同期间多条数据优先选择可验证的最新披露/更新版本。
- 保留被替代版本的存在信息，避免把重述差异当经营增长。
- 历史分析以当时 `notice_date` 可见版本为准，不能回填后续重述后声称模型当时已知。

## 单位与百分比

- 不根据数值大小猜元/万元/亿元；单位未知时保留原值并 warning。
- ROE、毛利率和同比通常输出 percentage points：`18.2` 表示 `18.2%`。
- 只有来源明确为 ratio 时才执行 `×100`；同一字段不得重复缩放。

## 质量门

| 检查 | 失败处理 |
| --- | --- |
| 根结构不是对象/数组 | error |
| 没有可识别 reports | error |
| report_date 缺失 | 保留行并 warning，不参与趋势排序 |
| 同期间重复 | warning，确定性去重 |
| 收入/利润非有限数 | 置 null 并 warning |
| 只有单期 | warning，不下长期趋势结论 |
| 单位或 data_type 不明 | warning，不做派生比较 |
